About Neerazhi

AI-powered smart drainage intelligence for urban flood resilience.

The problem

Tamil Nadu's cities — Chennai, Coimbatore, Madurai — face repeated urban flooding during the northeast monsoon. Drainage networks get overwhelmed not because of capacity alone, but because of poor real-time visibility. Maintenance teams have no way to see which manholes are filling up, which sections are blocked, or where water should be diverted — until it's already flooding.

What Neerazhi does

Neerazhi puts a live intelligence layer on top of the drainage network. Ultrasonic sensors in manholes report water levels every 2 seconds. A graph algorithm continuously computes the safest diversion route. And when operators need situational awareness fast, Claude AI reads all 15 sensors at once and gives a plain-language flood risk assessment with specific recommendations.

Why real AI, not fake AI

Most "AI dashboards" you'll find online use weighted formulas and call them AI. Neerazhi does that too for the fast, deterministic parts — risk scores, Dijkstra routing, DNVI calculations. But when it comes to synthesising 15 sensors into a human-readable assessment that accounts for context (rain status, diversion path, vulnerable nodes, local geography), that's genuinely where a language model adds value that a formula can't.

The Claude API receives the full sensor snapshot and returns analysis written by an actual language model — not a template with numbers plugged in.

Sensor hardware mapping

The 15 manhole nodes (MH1–MH15) are laid out in a 3×5 grid topology. Each node represents one physical manhole sensor. In real deployments, sensors publish to MQTT topics like neerazhi/sensors/MH1 with a JSON payload containing the water level reading.

See the MQTT integration docs for the exact payload format and topic structure.
