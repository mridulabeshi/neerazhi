Neerazhi docs

Everything you need to install, run, and extend the Neerazhi smart drainage system.

Requirements: Node.js 18+, npm 9+. An Anthropic API key for AI analysis. An MQTT broker only if using real sensors.

Installation

Clone or download the project, then install dependencies:

git clone https://github.com/yourname/neerazhi.git
cd neerazhi
npm install

Environment setup

Copy the example env file and fill in your values:

cp .env.example .env

Edit .env:

ANTHROPIC_API_KEY=sk-ant-...
PORT=3000
MQTT_BROKER=mqtt://localhost:1883
MQTT_TOPIC=neerazhi/sensors/#

MQTT_BROKER and MQTT_TOPIC are only required when using real sensors. In simulator mode they are ignored.

Running locally

# Production
npm start

# Development (auto-restart on changes)
npm run dev

Then open http://localhost:3000 in your browser.

Backend

The backend (server/index.js) does five things:

  • Serves static files from public/
  • Runs the sensor simulator tick every 2 seconds
  • Connects to an MQTT broker when in real-sensor mode
  • Pushes sensor updates to all connected browsers via WebSocket
  • Proxies requests to the Anthropic Claude API (so your API key stays server-side)

WebSocket events

Server → client

TypePayloadDescription
init{ data, rain, sensorMode }Full state on first connect
sensors{ data }Updated sensor array every tick
rainStatus{ rain }Rain toggle confirmed
modeStatus{ sensorMode }Mode switch confirmed

Client → server

TypePayload
toggleRain{ value: bool }
setMode{ mode: "sim" | "mqtt" }

GET /api/sensors

Returns the current sensor snapshot without opening a WebSocket.

curl http://localhost:3000/api/sensors
{
  "sensors": [
    { "id": 1, "level": 42, "predicted": 47, "riseRate": 1.2,
      "blockage": 8, "risk": 24, "status": "warning" },
    ...
  ],
  "rain": false,
  "sensorMode": "sim"
}

POST /api/analyse

Sends sensor data to Claude and returns an AI flood assessment.

curl -X POST http://localhost:3000/api/analyse \
  -H "Content-Type: application/json" \
  -d '{
    "sensors": [...],
    "rain": false,
    "path": "MH7 → MH8 → MH13",
    "dnvi": "MH8, MH9, MH13"
  }'
{ "analysis": "Current drainage levels are moderate..." }

GET /api/history

Returns the last 20 critical alerts (in-memory, resets on server restart).

Simulator mode

The default mode. Every 2 seconds, water levels increase randomly when rain is on, and decrease slowly when off. Blockage scores are randomised on startup. No hardware needed.

MQTT real sensors

Switch to "Real MQTT" mode in the dashboard toolbar. The server will connect to the broker defined in .env and subscribe to MQTT_TOPIC.

Each sensor node should publish to a topic like neerazhi/sensors/MH1.

MQTT payload format

Publish JSON with nodeId (1–15) and level (0–100):

// Topic: neerazhi/sensors/MH1
{
  "nodeId": 1,
  "level": 54.3
}

The server clamps level to 0–100 and updates that node's entry in the shared state.

Deploy to Railway

npm install -g @railway/cli
railway login
railway init
railway up

Set ANTHROPIC_API_KEY as an environment variable in the Railway dashboard under Variables.

VPS with nginx + pm2

# Install pm2 and start the app
npm install -g pm2
pm2 start server/index.js --name neerazhi
pm2 save

# nginx config (reverse proxy)
server {
    listen 80;
    server_name yourdomain.com;
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}

The Upgrade and Connection headers are required for WebSocket proxying.
